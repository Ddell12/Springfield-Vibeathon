## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can demo what you've built.
2. **Infrastructure before features.** The first phase sets up everything so feature work in later phases is fast and clean.
3. **Magic moment first.** The core value proposition — parent describes a tool, AI builds it — works as early as possible.
4. **Test as you go.** Vitest unit tests for components, Playwright E2E for critical flows. Each phase includes verification.
5. **Progressive enhancement.** Start with the simplest working version, then layer on polish.
6. **Review before proceeding.** Push each completed phase as a PR. GitHub Actions runs tests + lint. CodeRabbit reviews code.
7. **Config-based, not code-based.** The AI generates JSON configurations, not arbitrary code. Pre-built React components handle rendering.
8. **Auth last.** Auth is deliberately deferred to the final phase. Build and test everything without login flows first.
9. **Use proven libraries.** dnd-kit for drag & drop, motion for animations, Convex Agent for chat, convex-helpers for utilities. See `docs/architecture/dependencies.md`.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

1. **Start a session:** Give your coding agent the phase prompt at the beginning of each phase.
2. **Read selectively:** Each phase lists specific focused files to read. Load only what's listed.
3. **Let it work:** The agent reads the roadmap, finds the first unchecked task, implements it, marks it complete.
4. **One session = one phase (ideally):** Complete a full phase per session. Resume from last unchecked task if interrupted.
5. **Push a PR for review:** GitHub Actions runs tests automatically. CodeRabbit reviews code. Don't start next phase until PR is merged.
6. **Run tests after each phase:** `npm test` for unit tests, `npx playwright test` for E2E.

### Session Tips

- **Don't read everything:** Each phase lists specific focused files. Load only what's listed.
- **Don't skip tasks:** Tasks are ordered for dependency resolution.
- **Verify after each phase:** Run the app + tests before moving on.
- **Update the status line:** `**Status:** X/62 tasks complete` and `**Current Phase:** Phase N`.

### Prompt Templates

**Starting a new phase:**
> "Read docs/product-roadmap.md and find the current phase. Read only the Reference files listed for that phase. Start on the first unchecked task. After each task, mark it `[x]` in the roadmap. Continue through the phase."

**Resuming after a break:**
> "Read docs/product-roadmap.md. Find the first unchecked task. Read the Reference files for the current phase. Continue."

**After completing a phase:**
> "Phase [N] is complete. Run `npm test -- --run` and `npx playwright test`. If all pass, create branch `phase-{N}/{slug}`, commit, push, open PR targeting main."

**Fixing an issue:**
