Dual Architecture Note
Important
Two builder architectures coexist
builder (v1) uses Convex Agent + config-based tool generation (5 tool types, JSON configs rendered by React components). builder-v2 uses E2B sandboxes + Claude code generation (FragmentSchema, arbitrary React apps). These represent different product strategies — v1 is the original CLAUDE.md design, v2 is the Vibeathon pivot. Both share the same Convex backend but have separate chat flows, schemas, and rendering approaches.

Purpose & Boundaries

The therapy-focused AI app builder that represents the Vibeathon pivot. Users describe therapy tools in plain language; Claude generates complete React applications rendered in E2B cloud sandboxes. This slice owns the chat interface, code generation prompts, E2B sandbox lifecycle, and live preview iframe. It does NOT own the Convex backend CRUD (that's in convex/projects.ts) or the config-based therapy tool components (those belong to therapy-tools).
