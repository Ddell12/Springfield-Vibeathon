## Summary

This spec documents design decisions to maximize open-source library usage and minimize custom code for the Bridges hackathon build. It modifies the existing roadmap (`docs/product-roadmap.md`) with library substitutions, scope trim, Google Stitch for visual design, and Zod for runtime validation.

**Core principle:** Every line you don't write is a line you don't debug.

TASK-023 and TASK-024 should be annotated as stretch goals, not removed.

**@assistant-ui/react note:** This is a pre-1.0 library (0.12.x) with a fast-moving API. Verify the `ExternalStoreRuntime` and `convertMessage` patterns against current docs before implementation.

## Architecture: Three Layers

Bridges uses a three-layer architecture that separates concerns and maximizes library reuse:

### Visual Layer — Google Stitch
Google Stitch generates all visual UI: page layouts, app shell, therapy tool component visuals, landing page, templates gallery, My Tools page, share dialog. Output is React components styled to match the design tokens in `docs/design/design-tokens.md`.

### Behavior Layer — Open-Source Libraries
Interactive behavior is handled by libraries wired into Stitch-generated visuals:

| Library | Version | Purpose |
|---------|---------|---------|
| `@assistant-ui/react` | 0.12.x | Chat UI with ExternalStoreRuntime for Convex Agent |
| `@dnd-kit/react` | 0.3.x | Drag-to-reorder on visual schedule and communication board |
| `motion` | 12.x | Animations (token earn, card select, phase transitions) |
| `zustand` | 5.x | Tool interaction state (token counts, selections) |
| `use-sound` | 4.x | TTS audio playback, iOS Safari safe |
| `usehooks-ts` | 3.1.x | useMediaQuery, useCountdown, useCopyToClipboard, useDebounce |
| `react-error-boundary` | 6.1.x | Declarative error boundaries with retry/reset |
| `react-markdown` | 10.1.x | Markdown rendering in chat (backup if assistant-ui doesn't cover it) |

### Data Layer — Convex + AI
Server-side data and AI processing:

| Component | Purpose |
|-----------|---------|
| Convex | Database, real-time subscriptions, file storage |
| `@convex-dev/agent` | Chat threads, streaming, tool calling, React hooks |
| `@convex-dev/rag` | Embeddings, chunking, vector search for therapy knowledge |
| Claude Sonnet via `@ai-sdk/anthropic` | LLM for interpreting descriptions and generating tool configs |
| Google `gemini-embedding-001` via `@ai-sdk/google` | 768-dim embeddings for RAG |
| ElevenLabs | Text-to-speech for communication boards |
| Google Gemini / fal.ai | AI image generation for picture cards |
| Zod | Runtime validation of tool configs generated by Claude |

## Chat UI Architecture

The chat interface uses `@assistant-ui/react` with Convex Agent via the ExternalStoreRuntime adapter pattern.

### Data flow

```
Convex Agent (useUIMessages) → ExternalStoreRuntime → @assistant-ui/react components
```

### Implementation (~40 lines)

Single file: `src/features/builder/components/chat/bridges-chat.tsx`

```typescript
import { useExternalStoreRuntime, AssistantRuntimeProvider } from "@assistant-ui/react";
import { Thread } from "@assistant-ui/react/ui";
import { useUIMessages } from "@convex-dev/agent/react";

function BridgesChat({ threadId }) {
  const { messages, isRunning } = useUIMessages(threadId);
  const sendMessage = useMutation(api.chat.initiateStreaming);

  const runtime = useExternalStoreRuntime({
    messages,
    isRunning,
    convertMessage: (msg) => ({
      role: msg.role,
      content: [{ type: "text", text: msg.text }],
      id: msg._id,
    }),
    onNew: async (message) => {
      await sendMessage({ threadId, content: message.content[0].text });
    },
  });

  return (
    <AssistantRuntimeProvider runtime={runtime}>
      <Thread />
    </AssistantRuntimeProvider>
  );
}
```

### What assistant-ui provides for free
- Message rendering with user/assistant styling
- Streaming text animation
- Markdown rendering (bold, lists, code blocks)
- Auto-resizing text input
- Scroll-to-bottom button
- Copy button on messages
- Empty/welcome state
- Accessible (ARIA, keyboard navigation)

### Theming
Customize `aui-*` CSS classes in `globals.css` to match design tokens (deep teal for user messages, light gray for assistant).

### Impact on roadmap
TASK-015 changes from "Build 3 chat components from scratch" to "Wire @assistant-ui/react to Convex Agent ExternalStoreRuntime and theme to match design tokens."

## Zod for Runtime Validation

Tool configs use Zod schemas as the single source of truth for both runtime validation and TypeScript types.

### Pattern

```typescript
// Define Zod schema
export const CommunicationBoardSchema = z.object({
  type: z.literal("communication-board"),
  title: z.string(),
  cards: z.array(z.object({
    label: z.string(),
    imageUrl: z.string().optional(),
    icon: z.string().optional(),
  })).min(1).max(20),
  sentenceStarter: z.string().default("I want"),
});

// Infer TypeScript type from schema
export type CommunicationBoardConfig = z.infer<typeof CommunicationBoardSchema>;
```

### Where Zod is used
1. **Tool config schemas** (`src/features/therapy-tools/types/tool-configs.ts`) — single source of truth
2. **Tool renderer** — `safeParse` before rendering, graceful fallback on invalid config
3. **Convex Agent tool definitions** — schema descriptions for Claude's tool calling
4. **Env validation** — already planned via `@t3-oss/env-nextjs`

### Impact on roadmap
TASK-011 changes from "write TypeScript interfaces" to "write Zod schemas with `z.infer<>` for inferred types."

## Scope Trim: 5 → 3 Therapy Tools

### Core set (must ship)

| Tool | Demo role | Libraries used | Custom code |
|------|-----------|---------------|-------------|
| **Communication Board** | Star of demo — TTS, picture cards, sentence building | Stitch visual, dnd-kit, use-sound, motion, next/image | Sentence builder (~30 lines) |
| **Token Board** | Universal ABA tool, visual impact | Stitch visual, motion, zustand | Earn/reset state (~15 lines) |
| **Visual Schedule** | Daily routine tool, drag-to-reorder | Stitch visual, dnd-kit, motion | Config → props mapping (~10 lines) |

**Total custom code across all 3 tools: ~55 lines.**

### Stretch goals (if time allows, ~1 hour each)

| Tool | Implementation |
|------|---------------|
| **Choice Board** | Stitch visual + shadcn ToggleGroup + motion |
| **First-Then Board** | Stitch visual + motion + usehooks-ts useCountdown |

### Rationale
- 3 polished tools > 5 rough tools for a hackathon demo
- Choice board is functionally a simplified communication board
- First-then board is the simplest tool — easy to add later
- The AI agent's tool definitions still include all 5 types — if a user asks for a choice board, the AI can explain it's coming soon

### Component build workflow
1. Design visual states in Google Stitch (empty, active, completed)
2. Export as React components
3. Wire in library hooks (dnd-kit, motion, zustand)
4. Connect to Convex tool config via `useQuery`
5. Validate config with Zod `safeParse` before rendering
