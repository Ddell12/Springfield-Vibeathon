# Bridges ‚Äî Contributing Guide

## What Is This Project?

Bridges is an AI-powered therapy app builder where ABA therapists, speech therapists, and parents of autistic children describe therapy tools in plain language and get working, interactive apps built by AI. Think "Replit for therapy tools" ‚Äî except the AI already speaks therapy language.

## Stack

- **Frontend:** Next.js 16 (App Router) + shadcn/ui + Tailwind v4
- **Auth:** Clerk v7 (`@clerk/nextjs`) ‚Äî sign-in/sign-up, JWT sessions, Convex integration via `ConvexProviderWithClerk`
- **Backend:** Convex (real-time, TypeScript, built-in vector search)
- **AI Code Generation:** Claude Sonnet via `@anthropic-ai/sdk` ‚Äî streaming SSE pipeline
- **Knowledge Base:** Google Gemini embeddings (768-dim) via `@convex-dev/rag`
- **Image Generation:** Google Gemini (`gemini-3-pro-image-preview`) with prompt caching
- **TTS:** ElevenLabs (`eleven_flash_v2_5`) with child-friendly voices
- **STT:** ElevenLabs Scribe v2
- **Preview:** Server-side Parcel bundling ‚Üí sandboxed iframe (WAB scaffold with Tailwind + 40+ shadcn components)
- **Deploy:** Vercel (app) + Vercel Deploy API (published apps)

## Architecture

### Vertical Slice Architecture (VSA)

- `src/core/` ‚Äî Universal infrastructure (providers, utils). Shared across all features.
- `src/shared/` ‚Äî Code used by **3+ features**. shadcn/ui primitives live here.
- `src/features/{name}/` ‚Äî Self-contained feature slices. All components, hooks, types, and logic for a feature in one directory.
- `src/app/` pages are **thin wrappers** (< 20 lines) that import from features.
- `convex/schema.ts` is the single schema file. `convex/{domain}.ts` files organize functions by domain.

### Streaming Code Generation Pipeline

Bridges generates complete React applications through a streaming SSE pipeline:

```
User describes app ‚Üí Claude streams React code via SSE with tool calls (write_file, generate_image, generate_speech)
  ‚Üí Files written to server-side WAB scaffold (Vite + React + Tailwind + 40+ shadcn components)
  ‚Üí Parcel bundles into a single self-contained HTML file
  ‚Üí Bundle sent to browser, rendered in sandboxed iframe via blob URL
  ‚Üí Persisted to Convex, publishable to Vercel
```

Key files:
- `src/app/api/generate/route.ts` ‚Äî SSE streaming endpoint with multi-turn tool loop
- `src/features/builder/lib/agent-prompt.ts` ‚Äî Agent system prompt with component library and design rules
- `src/features/builder/hooks/use-postmessage-bridge.ts` ‚Äî TTS/STT bridge between parent and iframe
- `src/core/providers.tsx` ‚Äî Clerk + Convex provider bridge
- `convex/auth.config.ts` ‚Äî Clerk JWT verification for backend auth
- `convex/sessions.ts` ‚Äî Session state machine
- `convex/image_generation.ts` ‚Äî Therapy image generation with caching
- `convex/aiActions.ts` ‚Äî TTS action with voice caching
- `convex/publish.ts` ‚Äî Vercel publish pipeline

## Code Conventions

### Convex
- All functions must be named exports (never default export)
- Use `v` validators on all args
- Actions need `"use node";` for external API calls ‚Äî keep them in separate files from queries/mutations
- Queries use `.withIndex()` ‚Äî never `.filter()`
- Index naming: `by_fieldName`
- Config at `convex/convex.config.ts` (inside `convex/`, not project root)

### Next.js
- App Router only (`src/app/`)
- Server Components by default ‚Äî `"use client"` only for hooks/interactivity
- Metadata via `export const metadata`, never JSX `<title>`

### Tailwind v4
- Config via `@theme` in `src/app/globals.css` ‚Äî no `tailwind.config.js`
- Use `cn()` from `src/core/utils.ts` for class merging
- Mobile-first: base styles for mobile, `md:` / `lg:` for larger screens

### shadcn/ui
- Components in `src/shared/components/ui/`
- Add via `npx shadcn@latest add <name>`
- Use semantic tokens: `bg-background`, `text-foreground`, `border-border`

### Design System
- Dual fonts: **Manrope** (headlines) + **Inter** (body)
- No 1px borders for sectioning ‚Äî use tonal background shifts instead
- Primary CTA: gradient from `#00595c` to `#0d7377` at 135deg
- All animations use `cubic-bezier(0.4, 0, 0.2, 1)`, minimum 300ms duration

## Terminology

- **app** ‚Äî what users build (never "tool" or "component" in the UI)
- **session** ‚Äî a build session with state machine (not "project")
- **blueprint** ‚Äî structured therapy-specific PRD generated by the AI, approved by the user before building

## What NOT to Do

- Don't expose developer jargon in the UI (no "component", "API", "deploy", "database")
- Don't use `style={{}}` when Tailwind has an equivalent
- Don't put `"use node";` in files that export queries or mutations

## Authentication

Clerk v7 handles all user authentication. The integration spans three layers:

- **Frontend:** `ClerkProvider` wraps the app in `src/app/layout.tsx`. `ConvexProviderWithClerk` in `src/core/providers.tsx` bridges Clerk sessions to Convex. `UserButton` appears in the sidebar and header.
- **Pages:** `/sign-in` and `/sign-up` use Clerk's pre-built components.
- **Backend:** `convex/auth.config.ts` verifies Clerk JWTs using the issuer domain.

Unauthenticated users can explore templates and use the builder. Protected routes (like My Apps) require sign-in.

### Test User Sign-In (Headless Browsers)

Password auth fails in headless browsers (Clerk flags new devices). Use the **email code flow**:

1. Go to `/sign-in`, enter `e2e+clerk_test@bridges.ai`, click Continue
2. Click **"Use another method"** ‚Üí **"Email code"**
3. Enter code **`424242`** (always works for `+clerk_test` emails in dev mode)

The `+clerk_test` suffix is a Clerk dev feature: no real email sent, code is always `424242`.

- `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` ‚Äî Clerk public key
- `CLERK_SECRET_KEY` ‚Äî Clerk server-side key
- `ANTHROPIC_API_KEY` ‚Äî Claude API key (for streaming route)

### Convex Dashboard
- `ANTHROPIC_API_KEY`
- `CLERK_JWT_ISSUER_DOMAIN` ‚Äî Clerk JWT issuer for auth verification
- `GOOGLE_GENERATIVE_AI_API_KEY` ‚Äî for RAG embeddings and image generation
- `ELEVENLABS_API_KEY` ‚Äî for TTS and STT
- `VERCEL_TOKEN`, `VERCEL_PROJECT_ID` ‚Äî for publish pipeline
